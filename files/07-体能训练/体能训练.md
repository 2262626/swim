# 体能训练模块说明

## 一、模块目标

本模块用于新增一个独立于游泳分析之外的“固定动作体能训练姿势识别”路由，定位为：

- 面向固定器械的姿势识别与动作纠正
- 面向 6-18 岁青少年与 18 岁以上成年人两套提示风格
- 仅前端本地运行，不依赖后端
- 同时支持实时分析与本地视频分析
- 优先适配手机与 iPad 场景，不要求 PC 页面体验

当前代码已经按这个方向完成了首批实现，入口路由为 `/fitness`。

## 二、当前实现与代码映射

当前实现由以下前端文件组成：

- `src/router/index.js`
  - 新增 `/fitness` 路由
- `src/views/FitnessTrainingView.vue`
  - 体能训练识别页面
  - 包含实时分析、视频分析、开始分析、暂停/继续、导入视频、语音开关、器械切换、年龄段切换
- `src/composables/useFitnessAnalysis.js`
  - 固定器械分析逻辑
  - 视频质量门控
  - 动作阶段判定
  - 次数统计
  - 分项评分
  - 双年龄段语音提示词
- `src/components/TopBar.vue`
  - 支持从游泳模块切换到体能模块

## 三、当前产品边界

### 1. 当前已经支持

首批只支持 5 个“前端骨骼判定确定性较高”的动作：

1. 史密斯深蹲
2. 坐姿腿屈伸
3. 坐姿划船
4. 龙门架夹胸
5. 罗马椅挺身

选择这 5 个动作的原因是：

- 关键点暴露相对充分
- 动作轨迹较稳定
- 单视角下更容易建立阶段状态机
- 对 MediaPipe 33 点骨骼的可解释性较高
- 能够在不接后端的前提下做出较可靠的 MVP

### 2. 当前不建议一期接入

以下动作在原始素材里有，但当前不建议一期直接接入：

1. 上斜哑铃卧推
   - 卧姿时身体被凳子遮挡较多
   - 哑铃轨迹会遮挡腕、肘、肩关键点
   - 手机机位稍有偏差就容易误判
2. 平板卧推
   - 卧姿遮挡更明显
   - 杠铃和保护架会挡住上肢关键点
   - 如果未来要做，需要把安全监控与失败检测一起设计
3. 杠铃片取放流程
   - 这更像“器械使用规范识别”，不是稳定重复动作
   - 流程节点多，单纯骨骼阈值不够
4. 引体向上 / 双杠臂屈伸
   - 悬垂动作极易出现手腕、肩部被横杆遮挡
   - 动作的正确性依赖更强的时序特征
   - 对摄像头安装位置要求很高
5. 坐姿腿弯举
   - 与腿屈伸相比，需要更稳定的侧视角
   - 器械挡板和小腿滚轴容易遮挡关键点
   - 可以放到二期，与腿屈伸共用部分状态机再扩展

## 四、当前交互设计

当前页面交互重点如下：

- 页面默认进入预览态，不自动开始分析
- 用户点击“开始分析”后才进入分析循环
- 可以切换：
  - 实时分析
  - 视频分析
- 视频分析使用本地导入文件，不上传服务端
- 支持切换前后摄像头
- 支持切换器械
- 支持切换年龄组
- 支持语音开关
- 控制面板改为小图标展开/收起，减少视频区遮挡
- 控制面板已拆分为总览 / 模式 / 器械 / 提示 / 检查多个分区，并支持快捷入口
- 当前语音仍基于浏览器 TTS，但会自动优先选择可用中文系统音色
- 支持按器械显示测试辅助线，优先根据人物骨骼动态更新，并可手动开关
- 6-18 岁模式保留独立语音提示，但不再额外显示顶部绿色提示条
- 支持动作阶段、评分、次数、质量、置信度、FPS、检查项展示

这符合固定硬件机位的后续落地方向：设备安装好后，用户只需要对准器械站位并点击开始。

## 五、双年龄段提示词策略

当前不是简单更换一句称呼，而是两套提示风格：

- 青少年
  - 用词更简单
  - 更偏鼓励式和具象化
  - 例如“像坐小椅子一样慢慢蹲下去”
- 成年人
  - 用词更技术化
  - 更强调核心、躯干、髋膝踝、离心控制等术语
  - 例如“保持核心收紧和髋部后坐”

这样做的好处是：

- 同一算法可以服务两类人群
- 家长、教练和成人训练者都更容易接受提示语
- 后续如果接 TTS 音色，也可以按年龄组继续细分

## 六、当前判定方式

当前判定不是黑盒大模型，而是“通用骨骼 + 器械规则状态机”：

1. 先检查当前帧是否具备判定条件
2. 再按器械进入对应状态机
3. 根据关键角度、轨迹、对称性、稳定性做分项判断
4. 计算评分与提示词
5. 在达到完整动作周期后计数

这套方式适合固定器械动作，因为动作轨迹更固定，阈值法比游泳场景更可控。

## 七、视频质量门控

为了避免“人还没准备好就开始乱判”，当前实现加入了证据门控。只有满足这些条件才进入正式判定：

- 分辨率至少约 640x480
- 有效关键点数量足够
- 平均骨骼置信度足够
- 当前器械所需关键点至少达到 75% 可用
- 人物在画面中占比不能太小
- 人物不能贴边站位

如果不满足条件，页面会返回“证据不足 / 当前帧不具备判定条件”，而不是直接输出错误纠正建议。

## 八、适用场景

当前版本适合这些场景：

- 健身房固定器械教学
- 青少年力量训练辅助
- 体测/训练营现场纠正
- 固定机位回放复盘
- 教练辅助评分而非正式裁判

## 九、验收建议

一期建议按以下标准验收：

1. 五个动作都能正常进入 `/fitness` 路由分析
2. 实时分析与视频分析都能正常跑通
3. 切换器械后会清空上一个动作的状态和计数
4. 人未完整入镜时，系统不应强行输出“动作错误”
5. 双年龄段语音提示能随动作问题变化
6. 在手机和 iPad 上操作区不溢出、不遮挡主要画面

## 十、下一步建议

如果要继续做二期，建议优先顺序如下：

1. 增加坐姿腿弯举
2. 增加 Pec Fly / 胸推类固定轨迹器械
3. 增加测试视频批量回放记录表
4. 增加“示范视频 + 实时识别”双屏模式
5. 增加动作合格阈值配置面板

如果要做三期，再考虑：

- 卧推类
- 引体 / 双杠
- 器械安全流程识别
- 多机位或固定硬件盒子部署
