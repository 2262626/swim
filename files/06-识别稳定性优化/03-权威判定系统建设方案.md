# 03-权威判定系统建设方案

> 更新时间：2026-03-30
> 用途：用于向老板或客户说明“为什么当前系统会误判”“如果要做权威判定系统应该怎么做”“纯前端是否足够”“有哪些替代方案和预期效果”。

---

## 1. 先说结论

当前系统已经能做：

- 骨骼识别
- 泳姿和阶段的基础判断
- 自动暂停
- 纠错建议
- 教练辅助分析

但当前系统还不能直接宣称为：

- 权威判定系统
- 可直接输出“合格 / 不合格”的最终裁决系统

原因不是单一的 `@mediapipe/pose` 版本问题，而是以下三类问题叠加：

1. 当前模型是通用人体姿态模型，不是游泳专项判定模型。
2. 当前规则对起跳、过渡、水下、遮挡等阶段的时序门控还不够强。
3. 游泳场景天然存在低画质、角度遮挡、水花干扰等高噪声问题。

一句话给老板：

- 如果目标只是“训练辅助与纠错”，当前路线可继续增强。
- 如果目标是“权威判定”，必须升级成“标准化采集 + 后端时序模型 + 质量控制 + 证据不足拒判”的系统，而不是只靠前端单视频即时判断。

## 1.1 当前代码已落地的 Phase 1 范围

截至 2026-03-30，这一轮代码已经先把“稳定版教练辅助系统”的第二层打下来了，当前前端已落地：

- `PREP / START / UNDERWATER / TRANSITION / TURN / FINISH / SWIM_ACTIVE / INVALID` 分阶段状态机
- 自由泳 / 仰泳 / 蛙泳 / 蝶泳分别计数的状态机
- 视频质量门控
- 证据不足拒判
- 机位区分配置（池边侧拍 / 斜侧拍 / 正前正后拍 / 水下侧拍）
- 真实视频回放验证表模板

这一轮额外补上的保守策略是：

- 起跳准备姿态不评分、不自动暂停
- 水下滑行、转身、结束阶段不评分、不自动暂停
- 只有 `SWIM_ACTIVE` 才允许评分、抓拍、自动暂停、计数汇总
- 当泳姿识别不稳定而回退到“目标泳姿”时，不触发自动暂停
- 底部主状态直接显示“准备阶段 / 起跳阶段 / 水下阶段 / 过渡阶段 / 转身阶段 / 结束阶段 / 证据不足”
- 轻量提示改成页面最右侧浮层，3 秒自动消失，避免持续干扰

还没有完成的部分包括：

- 基于真实游泳视频数据集的误判率验证
- 真正面向“权威判定”的后端时序模型

## 2. 当前误判的根因

你现在遇到的现象是：

- 学员明明还在准备起跳
- 系统却已经给出“动作异常 high，评分=50”

这说明现在的问题不是“骨骼拿不到”，而是“拿到骨骼后，系统过早开始打分”。

当前根因可以拆成三层：

### 2.1 阶段门控不够严格

现在系统虽然已经有 `PREP / TRANSITION / SWIM_ACTIVE` 的思路，但还不够“裁判级”。

真正要做权威判定，必须把动作状态机再细化为：

- `PREP`：准备姿势
- `START`：起跳发力
- `UNDERWATER`：入水后水下阶段
- `SURFACE_TRANSITION`：出水过渡阶段
- `SWIM_ACTIVE`：有效划水阶段
- `TURN`：转身阶段
- `FINISH`：结束阶段
- `INVALID`：证据不足 / 不能判定

只有在 `SWIM_ACTIVE` 时才允许：

- 泳姿评分
- 动作异常判定
- 自动暂停
- 划臂计数

### 2.2 当前没有“拒判机制”

现在系统更像是：

- 只要拿到骨骼，就尽量给一个判断

但权威系统必须有一条硬规则：

- 当证据不足时，不给结论

也就是要支持：

- `无法判断`
- `证据不足`
- `当前阶段不评分`

对于老板和客户，这一点非常关键：

- 真正专业的系统，不是“什么都敢判”
- 而是“能判的时候才判，不能判的时候明确拒判”

### 2.3 当前评分是规则评分，不是专项判定模型

目前系统是：

- 通用姿态关键点
- 前端阈值规则
- 等权打分

这很适合做：

- 提醒
- 纠错
- 辅助教学

但不适合直接做：

- 高置信度最终判定

## 3. 如果要做“权威判定系统”，建议分三条路线看

## 3.1 方案A：增强版纯前端

### 适用目标

- 快速止损
- 让当前产品明显变稳
- 继续保留“前端实时识别”的体验

### 应做内容

1. 补强阶段状态机
   - 把 `PREP / START / UNDERWATER / SURFACE_TRANSITION / SWIM_ACTIVE / TURN / FINISH / INVALID` 做完整
   - 非 `SWIM_ACTIVE` 阶段一律不评分、不报警、不计数

2. 增加“证据不足拒判”
   - 画质差
   - 关键点可见度不足
   - 遮挡严重
   - 角度不合适
   - 水花遮挡过重
   在这些情况下，直接显示：
   - `当前阶段不评分`
   - `证据不足，无法判定`

3. 改造自动暂停策略
   - 仅在 `SWIM_ACTIVE` 触发
   - 必须满足连续 N 帧稳定命中
   - 需要增加“置信度 + 时序一致性 + 画质质量”三重门控

4. 按泳姿分别做计数状态机
   - 自由泳 / 仰泳：交替周期
   - 蛙泳：手腿时序
   - 蝶泳：双臂同步周期

5. 加入前端视频质量检测
   - 分辨率不足
   - 人体过小
   - 运动模糊
   - 逆光 / 曝光过低
   - 遮挡率过高
   一旦不达标，提示重新拍摄或转离线分析

### 效果预期

- 可以明显减少“准备起跳就报警”
- 可以显著降低误判与误暂停
- 适合做更成熟的教练辅助系统

### 局限

- 仍然不是权威判定系统
- 单机位、单视频、通用模型的天花板依然存在
- 不适合向客户承诺“稳定专业判定准确率”

### 适合对老板怎么说

- 这是最低成本、最快见效的止损方案
- 能把产品从“容易误判”提升到“可用于稳定辅助教学”
- 但还不能叫“权威判定系统”

---

## 3.2 方案B：前端采集 + 后端时序模型判定

### 适用目标

- 真正往“专业级判定”靠近
- 对外能更稳地交付“训练分析”能力
- 是我最推荐的路线

### 核心架构

前端负责：

- 相机 / 视频采集
- 基础姿态提取
- 实时预览
- 轻量提示
- 低风险自动暂停

后端负责：

- 时序分段
- 动作阶段识别
- 异常识别
- 评分校准
- 置信度估计
- 拒判机制

### 应做内容

1. 前端不再直接输出最终裁决
   - 前端只输出“候选阶段 / 候选异常 / 候选快照”

2. 后端新增时序模型
   - 输入：关键点序列 + 置信度 + 可选视频特征
   - 模型可选：TCN / LSTM / Transformer
   - 输出：
     - 阶段标签
     - 异常标签
     - 评分
     - 置信区间
     - 是否拒判

3. 做泳姿专项标注数据集
   - 先从 4 泳姿中选 1 到 2 个最常用泳姿做 MVP
   - 先做“是否进入有效划水阶段”
   - 再做“异常分类”
   - 最后再做“评分”

4. 做质量控制模型
   - 视频质量评分
   - 关键点质量评分
   - 遮挡评分
   - 拍摄角度分类

5. 按机位建模
   - 池边侧拍
   - 正前 / 正后
   - 水下
   - 高机位
   不同机位的规则、阈值、模型不能混用

### 效果预期

- 这条路线最有机会做出“客户能认可”的专业系统
- 能显著提升准备阶段、起跳、转身、水下阶段的稳定性
- 可以逐步接近“权威判定”

### 局限

- 成本明显高于纯前端
- 必须做数据标注
- 交付周期会拉长

### 适合对老板怎么说

- 如果公司要卖“专业游泳动作分析系统”，这条路线最值得投
- 它不是最便宜，但是真正有机会做成壁垒

---

## 3.3 方案C：多机位 / 水下相机 / 穿戴式传感器混合方案

### 适用目标

- 追求最高稳定性
- 面向专业队、训练机构、高单价客户

### 核心思路

不要把所有判断都压在“单个普通视频”上，而是引入更稳定的信号源：

- 侧上方固定机位
- 水下机位
- 头戴或腰部 IMU
- 智能泳镜 / 可穿戴设备

### 为什么这条路线更强

因为很多公开研究里，真正高准确率的系统都不是：

- “随手拿一个手机视频 + 浏览器前端 + 当场直接权威判定”

而是：

- 固定场景
- 更好的传感器
- 更强的时序信息
- 更可控的采集流程

### 效果预期

- 对泳姿类型、划频、圈数、起跳事件、转身事件会更稳定
- 对遮挡和水花更不敏感
- 最适合做高端版本

### 局限

- 成本最高
- 部署复杂
- 客户教育成本高

### 适合对老板怎么说

- 这条路线适合做高客单价专业版
- 不适合先拿来做大众化低成本交付

---

## 4. 推荐路线

如果你的目标是“现在要能给老板交差，同时后面真能做成”，我建议这样走：

### Phase 0：1 到 3 天，先止损

立即改掉当前最伤产品信任的问题：

1. 准备 / 起跳 / 水下阶段全部不评分
2. 自动暂停只允许在 `SWIM_ACTIVE`
3. 画质差、遮挡大、角度不合适时直接拒判
4. 页面显示：
   - `准备阶段`
   - `等待进入有效游泳动作`
   - `证据不足，当前不做评分`

这一阶段的目标不是提升上限，而是：

- 先把明显错误判定拿掉

### Phase 1：2 到 4 周，完成增强版纯前端

1. 分阶段状态机
2. 分泳姿计数状态机
3. 视频质量门控
4. 证据不足拒判
5. 机位区分配置

这一阶段的定位是：

- 稳定版教练辅助系统

### Phase 2：4 到 8 周，做后端时序分析 MVP

1. 固定一个泳姿先做
2. 固定一个机位先做
3. 建立标注规范
4. 训练阶段识别 + 异常识别模型
5. 输出置信度与拒判

这一阶段的定位是：

- 专业分析 MVP

### Phase 3：8 到 16 周，向权威判定系统推进

1. 扩展到更多泳姿
2. 扩展到多机位
3. 加入水下机位或 IMU
4. 做版本化评估体系
5. 建立客户可验收指标

这一阶段的定位是：

- 高可靠专业产品

## 5. 公开可参考的类似系统与公开指标

说明：

- 下面这些结果只能作为“可行性参考”
- 不能直接当成你们当前系统的可承诺准确率
- 因为任务范围、场景控制程度、数据量、机位、传感器都不同

### 5.1 游泳专用视频姿态估计

WACV 2018 的研究《Activity-conditioned continuous human pose estimation for performance analysis of athletes using the example of swimming》：

- 研究目标：在真实游泳视频中做泳姿相关姿态估计
- 结论：在游泳这种高挑战环境下，通过加入泳姿信息和连续视频信息，正确关节点检测比基线最多提升约 `+16%`

这说明：

- 游泳场景确实需要专项化建模
- 直接拿通用姿态模型硬上，效果上限有限

### 5.2 基于姿态序列的游泳事件识别

CVPRW 2020 的研究《Decoupling Video and Human Motion: Towards Practical Event Detection in Athlete Recordings》：

- 研究目标：基于 2D pose sequence 做游泳起跳事件检测
- 结果：在有限数据下，游泳起跳事件检测 `F1 > 91%`

这说明：

- 对“起跳 / 事件”这类时间边界明确的任务，姿态序列 + 规则 / 时序模型是有效的
- 也说明你当前“准备起跳就被按异常评分”这个问题，本质上应该先做阶段判定，不该先做异常评分

### 5.3 单视频 / 视频模型做泳姿识别

2025 年公开论文《Human swimming posture recognition combining improved 3D convolutional network and attention residual network》：

- 结果：识别准确率 `95%`
- Recall `93.26%`
- F1 `92.87%`

但要注意：

- 这是专项模型
- 是受控任务
- 不等于你们现在单前端规则系统的现状

### 5.4 可穿戴设备路线

2022 年《Automatic Swimming Activity Recognition and Lap Time Assessment Based on a Single IMU: A Deep Learning Approach》：

- 单个 IMU 即可把 8 类游泳活动分类做到整体 `F1 = 0.96`
- 时间精度达到 `0.02s`

2021 年《Is machine learning and automatic classification of swimming data what unlocks the power of inertial measurement units in swimming?》：

- 基于 IMU 的泳姿预测在实验中达到 `>= 0.99` 的 weighted precision / recall / F1 / accuracy

2023 年关于 FORM Goggles 的研究：

- 相比视频真值，泳姿类型识别率达到 `99.7%`

这说明：

- 如果客户真想要“稳定、可交付、可解释”的泳姿与计数结果
- 可穿戴设备路线往往比单视频前端路线更容易做出高稳定性

## 6. 针对你提出的三个现实问题，怎么做

## 6.1 问题一：视频画质直接影响识别度，而且不可控

### 可行做法

1. 建立采集 SOP
   - 最低分辨率
   - 最低光照
   - 人体占画面比例
   - 最低帧率
   - 禁止强逆光

2. 上线“视频质量评分”
   - 模糊度
   - 曝光
   - 人体大小
   - 抖动程度

3. 不达标视频直接拒绝判定
   - 允许上传
   - 但不能给“权威结论”

### 不可行幻想

- 靠前端后处理把严重模糊、严重压缩、严重逆光视频完全救回来

### 应该怎么和客户说

- 系统可以分析，但不是所有视频都具备判定条件
- 低质量视频将进入“仅回放参考 / 不出结论”模式

---

## 6.2 问题二：拍摄角度不同，导致部分肢体被遮挡

### 可行做法

1. 固定机位
   - 先只支持侧拍
   - 或者先只支持固定高机位

2. 按机位建立模型与规则
   - 侧拍一套
   - 水下一套
   - 正拍一套

3. 引入双机位
   - 侧上方 + 水下
   - 或侧拍 + 正拍

4. 被遮挡的肢体不要强判
   - 不是补一个平均点就当真
   - 而是输出 `该动作当前证据不足`

### 不可行幻想

- 用单机位、任意角度、任意拍法，还要求所有泳姿和所有关节都稳定判定

### 应该怎么和客户说

- 权威判定必须建立在标准拍摄位上
- 如果客户不愿意标准化采集，就只能交付“辅助分析”而不是“权威判定”

---

## 6.3 问题三：外界干扰因素，比如水花

### 可行做法

1. 做遮挡检测
   - 检测关键点连续丢失
   - 检测置信度突降
   - 检测局部运动异常

2. 做时间窗口平滑
   - 不是看单帧
   - 而是看连续窗口的一致性

3. 引入分割或人体区域检测
   - 人体区域先分出来
   - 再做姿态推断或时序建模

4. 高级方案是水下机位或 IMU

### 不可行幻想

- 纯靠单帧关键点，在强水花和强遮挡下保持专业级稳定性

### 应该怎么和客户说

- 水花是游泳识别的天然干扰项
- 系统可以尽量鲁棒，但不能承诺“任何场景都百分百准确”
- 真正专业场景建议增加标准机位或传感器

---

## 7. 怎么和老板、客户汇报最稳

建议直接分三档对外说法：

### 档位1：教练辅助版

对外承诺：

- 实时识别
- 自动暂停
- 纠错建议
- 动作趋势参考

不承诺：

- 权威判罚
- 强一致性分数

### 档位2：专业分析版

对外承诺：

- 标准机位下的阶段识别
- 专项动作分析
- 报告输出
- 证据不足拒判

前提：

- 固定拍摄规范
- 后端模型参与

### 档位3：高可靠判定版

对外承诺：

- 更高稳定性
- 更强可复盘能力

前提：

- 多机位或穿戴式传感器
- 标准化采集
- 付费更高

---

## 8. 我建议你现在怎么做

最实际的建议是：

1. 先不要向老板承诺“纯前端单视频即可做权威判定”
2. 先承诺“本周先把误判止住”
3. 对中期路线主推“前端采集 + 后端时序模型”
4. 对高端客户准备“多机位 / IMU / 智能泳镜”的增强方案

如果要一句最适合汇报的话，可以直接这么说：

> 当前系统适合做教练辅助分析，不适合直接做权威判定。要把它做成真正可交付的权威判定系统，必须增加阶段门控、证据不足拒判、后端时序模型和标准化采集；如果客户追求更高稳定性，还需要多机位或可穿戴传感器。
