# 04-真实视频回放验证表

> 更新时间：2026-03-30
> 用途：用于阶段门控、拒判机制、自动暂停和泳姿评分的真实视频回放验证。

---

## 1. 回放验证目标

这一张表不用于“证明模型绝对正确”，而是用于验证下面几件事是否达标：

- 准备起跳时不会过早评分
- 起跳后能进入 `START / UNDERWATER / TRANSITION / SWIM_ACTIVE`
- 转身和结束阶段不会继续按正常划水评分
- 证据不足时能明确拒判
- 自动暂停只在真正可判阶段触发

---

## 2. 建议视频池

建议第一批至少准备 24 段视频，按下面分桶：

| 维度 | 建议分布 |
| --- | --- |
| 泳姿 | 自由泳 / 蛙泳 / 蝶泳 / 仰泳，各 6 段 |
| 机位 | 池边侧拍 / 斜侧拍 / 正前正后拍 / 水下侧拍 |
| 画质 | 高清 / 一般 / 偏差 |
| 干扰 | 无水花 / 中等水花 / 强水花 |
| 阶段覆盖 | 准备 / 起跳 / 水下 / 有效划水 / 转身 / 结束 |

---

## 3. 执行步骤

1. 固定目标泳姿、机位配置、模型精度。
2. 用“视频导入”方式逐条回放样例视频。
3. 观察页面中的阶段、质量、规则来源、自动暂停表现。
4. 按下表记录“预期”和“实际”是否一致。
5. 对误判样例截图留档，进入下一轮阈值修正。

## 5. 误判类型枚举

填写“误判类型”时，统一使用下面这些词，便于后续统计：

- `early_score`：准备或起跳阶段过早评分
- `miss_underwater`：未识别到水下阶段
- `miss_turn`：未识别到转身阶段
- `miss_finish`：未识别到结束阶段
- `false_swim_active`：非有效划水阶段误判为 `SWIM_ACTIVE`
- `false_invalid`：本应可判却被拒判
- `false_pause`：不该暂停却触发自动暂停
- `miss_pause`：应该提示却没有触发自动暂停

## 7. 当前建议

第一轮回放时，先不要追求“评分多准”，优先看下面三件事：

- 阶段有没有提前切错
- 本该拒判时有没有硬判
- 自动暂停有没有打断错位置

只要这三件事先稳住，后面的评分和专项优化才有意义。
