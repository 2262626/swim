## 1. 当前实现定位

当前系统采用“前端本地识别 + 规则分析 + HTTP 上报”的方式运行：

```text
视频/相机输入
    |
    v
MediaPipe Pose（关键点）
    |
    v
前端分析引擎
    |
    |-- 泳姿识别
    |-- 阶段识别
    |-- 规则评分
    |-- 自动暂停
    |-- 教学帧提示
    |-- 抓拍回放
    |
    v
会话上报接口
```

当前不依赖后端做实时分析。

后端当前承担的职责主要是：

- 会话创建
- 数据接收
- 结果存储

## 4. 当前状态机

当前页面的实际状态流可以概括为：

```text
idle -> running -> paused_auto -> running -> completed
            |           |
            |           -> paused_manual
            |
            -> error
```

说明：

- `idle`：未开始分析
- `running`：持续分析与更新 UI
- `paused_auto`：触发自动暂停
- `paused_manual`：手动暂停
- `completed`：结束本次分析并上报

---

## 5. 当前自动暂停规则

当前前端已落地的触发来源包括：

1. `pose_lost`
2. `score_below_threshold`
3. `angle_out_of_range`
4. `timing_mismatch`

当前已落地的控制策略包括：

- 连续命中帧数阈值
- 同类事件最小间隔
- 低等级自动恢复
- 中高等级等待人工处理

当前未落地的规则包括：

- 基于完整轨迹线的暂停触发
- 基于教练人工勾画区域的暂停触发

## 7. 当前实现的边界说明

当前系统是：

- 教练辅助分析工具

当前系统不是：

- 游泳专项训练大模型
- 自动判罚系统
- 完整成长分析平台

因此本阶段文档应避免把以下内容写成已实现：

- 手动批注图层
- 前后帧对比勾画
- 独立报告页
- PDF 按钮全链路可用
- 长期成长闭环已上线
